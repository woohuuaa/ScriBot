## System Architecture

KDAI processes live parliamentary debate streams, transcribes them in real-time, and extracts
questions using AI. The system integrates all processing logic within the Laravel backend for
streamlined operation.

## Core Components

### Frontend (kdai-frontend)

- **Port**: 5173 (dev), 80 (production)
- **Purpose**: Vue 3 SPA for viewing debates and transcripts
- **Communication**: REST API + WebSocket (Laravel Reverb)
- **Tech**: Vue 3, TypeScript, Vite, Tailwind CSS
- **Routes**: Dutch language paths (/debat, /ministerie, /zaal)

### Backend (kdai-backend)

- **Port**: 8001 (Nginx proxy), 8000 (Laravel), 8081 (Reverb)
- **Purpose**: Laravel API server with integrated AI processing
- **Database**: PostgreSQL 18
- **Storage**: MinIO S3-compatible object storage
- **Cache/Queue**: Valkey (Redis 9.0)
- **WebSocket**: Laravel Reverb on port 8081, proxied via Nginx on port 8001
- **AI Processing**: Integrated LLM for question extraction

### ATTS Services

#### kdai-atts-csp (Continuous Stream Processor)

- **Port**: 7010
- **Purpose**: Real-time M3U8 stream transcription with direct audio streaming
- **Tech**: Python, FFmpeg, WhisperLive WebSocket integration
- **Process**:
  - Connects to M3U8 streams via FFmpeg
  - Extracts and converts audio to 16kHz mono float32 in real-time
  - Streams audio directly to WhisperLive via WebSocket (no file storage)
  - Receives transcription segments from WhisperLive
  - Buffers and deduplicates transcription segments
  - Sends completed segments to backend via Redis queue (`ts_segment_complete`)

#### WhisperLive Server

- **Port**: 9090
- **Purpose**: Real-time speech recognition
- **Tech**: OpenAI Whisper implementation
- **Features**:
  - Real-time audio processing
  - Multiple language support
  - Segment-based corrections

#### kdai-m3u8-emulator (Development Only)

- **Port**: 6969 (external), 2121 (internal)
- **Purpose**: Simulates parliamentary M3U8 streams for development
- **Tech**: Python, HLS streaming
- **Usage**: Development and testing environment only

## TanStack DB Data Flow

The frontend uses **TanStack DB** collections for local-first state management with bidirectional
synchronization. Collections provide reactive, type-safe data access with optimistic updates.

**Architecture:**

- **9 Active Collections**: Debates, questions, segments, departments, rooms, topics, parties,
  users, files
- **Separate QueryClient**: Collections use dedicated `QueryClient` (not VueQueryPlugin)
- **Two-way Sync**: Client mutations persist to API, server events sync via WebSocket

**Key Principles:**

- **Optimistic Updates**: UI changes instantly before API confirmation
- **Automatic Rollback**: Failed mutations revert automatically
- **Separate Write APIs**: `writeUpsert/Update/Delete` for server events only
- **Race Condition Prevention**: Initialize collection before WebSocket setup
- **Type Safety**: End-to-end TypeScript with schema validation

## Key Design Decisions

### Real-time Processing

- FFmpeg streams M3U8 audio directly to WhisperLive without intermediate storage
- Redis queues handle transcript delivery from CSP to backend
- Laravel Reverb broadcasts updates to frontend clients via WebSocket
- CSP buffers and deduplicates transcription segments

### Simplified Architecture

- Question extraction integrated within Laravel backend
- Direct streaming eliminates intermediate storage layers
- MinIO used for file management, not real-time audio
- Reduces deployment complexity

### Scalability Considerations

- Redis queues for reliable message delivery
- Stateless services enable horizontal scaling
- Database indexing optimized for time-based queries

## Interface Specifications

### Storage Interface (S3-Compatible)

- **Implementation**: MinIO server
- **API**: Amazon S3 REST API
- **Authentication**: AWS Signature Version 4
- **Features**: Bucket operations, object operations, presigned URLs

### LLM Interface (OpenAI-Compatible)

- **Implementation**: Ollama (primary), configurable external APIs
- **API**: OpenAI Chat Completions API
- **Models**: Any model with chat completion interface
- **Features**: Streaming responses, function calling, token tracking

### WebSocket Interface (Laravel Echo)

- **Implementation**: Laravel Reverb (Pusher protocol compatible)
- **Backend**: Runs on `0.0.0.0:8081`, proxied through Nginx
- **Frontend**: Connects to `ws://localhost:8001/app`
- **Path**: `/app` must match between `REVERB_PATH` (backend) and `VITE_WS_PATH` (frontend)
- **Authentication**: Token-based channel authorization via Sanctum
- **Features**: Real-time events, private channels, presence tracking

**Critical**: Frontend must specify `wsPath` in Echo config to connect to local Reverb. Without it,
the client falls back to Pusher's cloud service.
