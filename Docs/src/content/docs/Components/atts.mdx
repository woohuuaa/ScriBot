ATTS provides real-time audio transcription from live M3U8 streams using WhisperLive, with the CSP
service streaming audio directly to WhisperLive and transcriptions back to the Laravel backend via
WebSocket.

## Services

### CSP - Continuous Stream Processor

Streams live M3U8 audio to WhisperLive and processes real-time transcriptions.

**Key Functions:**

- Connects to live M3U8 streams
- Extracts and streams audio using FFmpeg (16kHz mono, float32)
- Streams audio to WhisperLive via WebSocket in real-time
- Buffers and deduplicates transcription segments
- Detects and removes duplicate segments with fuzzy matching
- Sends completed segments to backend via Redis queue

**Audio Processing:**

```bash
# FFmpeg streaming configuration
Format: f32le (32-bit float little-endian)
Sample Rate: 16000 Hz
Channels: 1 (mono)
Chunk Size: 16384 bytes
```

**Segment Buffer Configuration:**

```python
SegmentBuffer(
    similarity_threshold=0.85,  # Fuzzy matching for duplicates
    short_sentence_threshold=15,  # Min chars for valid segment
    segments_to_wait_for_corrections=1,  # Wait for corrections
    max_dedup_history=50  # Max segments in dedup cache
)
```

**Redis Queue Output to Backend:**

```json
{
  "debate_id": "uuid",
  "segment_id": "00_05_23_456",
  "transcript_content": "Completed transcription segment.",
  "is_realtime": true
}
```

**Queue Name**: `ts_segment_complete`

### WhisperLive - Real-time Transcription Service

Real-time speech recognition using OpenAI Whisper.

**Key Functions:**

- Receives audio stream from CSP via WebSocket
- Performs real-time speech-to-text transcription
- Uses Voice Activity Detection (VAD) for segment boundaries
- Sends transcription segments and corrections back to CSP
- Supports multiple languages with Dutch optimization

**Configuration:**

```bash
WHISPERLIVE_HOST=kdai-whisperlive
WHISPERLIVE_PORT=9090
```

**Session Setup (sent by CSP):**

```json
{
  "uid": "unique-session-id",
  "language": "nl",
  "task": "transcribe",
  "model": "turbo",
  "use_vad": true
}
```

## Backend Integration

The Laravel backend orchestrates ATTS services through Redis queues:

- **Control Commands**: Start/stop capture commands sent via `csp_queue`
- **Transcript Reception**: Receives real-time transcript segments from `ts_segment_complete` queue
- **Filtering**: Applies text cleaning and deduplication logic
- **Real-time Broadcasting**: Dispatches WebSocket events to frontend via Laravel Reverb
- **Error Handling**: Manages service failures via `ts_segment_complete` queue (segment_id="error")

## Real-time Flow Architecture

```
M3U8 Stream → FFmpeg (CSP) → WhisperLive → CSP Buffer → Redis Queue → Backend → Reverb → Frontend
                                                                           ↓
                                                                       Database
```

**Processing Characteristics:**

- Segments sent when WhisperLive marks them as `completed=True`
- Fuzzy matching for duplicate detection (threshold: 0.85)
