Vue 3 + TypeScript frontend for parliamentary debate transcription and question management.

## Technology Stack

- **Framework**: Vue 3 with Composition API
- **Language**: TypeScript
- **Build Tool**: Vite
- **Styling**: Tailwind CSS
- **HTTP Client**: Axios
- **Real-time**: Laravel Echo + Pusher.js
- **Routing**: Vue Router 4
- **UI Components**: Custom components with Tailwind
- **Port**: 5173 (dev), 80 (production via nginx)

### State Management

#### TanStack DB Collections

The frontend uses **TanStack DB** for local-first state management with automatic backend
synchronization. Collections provide reactive, type-safe data access with optimistic updates.

**Architecture:**

- Collections use dedicated `QueryClient` (distinct from VueQueryPlugin)
- 9 Active Collections: Debates, questions, segments, departments, rooms, topics, parties, users,
  files
- Hybrid sync: API mutations + WebSocket real-time updates
- Optimistic updates with automatic rollback on errors

**Core Collections:**

```typescript
// src/db/schema.ts
import { createCollection } from '@tanstack/db';
import { queryCollectionOptions } from '@tanstack/query-db-collection';

// Question collection with full CRUD
export const debateQuestionsCollection = createCollection(
  queryCollectionOptions({
    queryKey: ['debate-questions'],
    queryFn: async () => [], // Populated per-debate
    queryClient,
    getKey: (item: DebateQuestion) => item.id,

    // Persistence handlers for client mutations
    onUpdate: async ({ transaction }) => {
      for (const update of transaction.mutations) {
        await debateQuestionService.updateQuestion(update.key as string, update.modified);
      }
      return { refetch: false }; // WebSocket handles sync
    },
  })
);
```

**Usage Pattern:**

```typescript
// 1. Reactive queries with useLiveQuery
const questionsQuery = useLiveQuery(
  q =>
    q
      .from({ questions: debateQuestionsCollection })
      .where(({ questions }) => eq(questions.debate_id, debateId.value)),
  [debateId] // Dependency array
);

// 2. Optimistic mutations
debateQuestionsCollection.update(questionId, draft => {
  draft.text = newText; // UI updates immediately
});

// 3. Real-time sync setup
useDebateQuestionsSync(debateId); // WebSocket sync composable
```

**Sync Composables:**

- `useDebateQuestionsSync(debateId)` - Real-time question updates
- `useTranscriptSegmentsSync(debateId)` - Live transcript segments
- Collections initialized before WebSocket setup to prevent race conditions

**Characteristics:**

- Optimistic updates: UI reflects changes immediately with automatic rollback on failure
- Offline-first: Collections work offline and sync when reconnected
- Type safety: End-to-end TypeScript with schema validation
- Multi-client sync: WebSocket broadcasts keep all browsers synchronized

## Real-time Features

### WebSocket Integration

```typescript
// Laravel Echo setup for Reverb connection
import Echo from 'laravel-echo';
import Pusher from 'pusher-js';

window.Echo = new Echo({
  broadcaster: 'reverb',
  key: import.meta.env.VITE_REVERB_APP_KEY,
  wsHost: location.hostname,
  wsPort: 8001,
  wsPath: import.meta.env.VITE_WS_PATH || '/app',
  forceTLS: location.protocol.startsWith('https'),
  auth: {
    headers: {
      Authorization: `Bearer ${token}`,
    },
  },
});
```

**Critical**: `wsPath` must match the backend's `REVERB_PATH`. Without it, the client falls back to
Pusher's cloud service instead of your local Reverb server.

### Channel Subscriptions

```typescript
// Private debate channel
window.Echo.private(`debate.${debateId}`)
  .listen('transcript.created', event => {
    // Handle new transcript segment
  })
  .listen('question.extracted', event => {
    // Handle extracted questions
  })
  .listen('transcript.updated', event => {
    // Handle transcript corrections
  });
```

## Configuration

### Environment Variables

```bash
# Backend API
VITE_API_BASE_URL=http://localhost:8001

# WebSocket (Laravel Reverb)
VITE_REVERB_APP_KEY=kdai-app-key-123
VITE_REVERB_HOST=localhost
VITE_REVERB_PORT=8001
VITE_WS_PATH=/app
```

**Note**: In production, Nginx proxies both API and WebSocket traffic through port 8001. Reverb
itself runs on port 8081.

### Build Configuration

```typescript
// vite.config.ts
export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': '/src',
    },
  },
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8001',
        changeOrigin: true,
      },
    },
  },
});
```

**Note**: During local development with `npm run dev`, Vite's dev server proxies API requests to the
backend nginx on port 8001. In production, the frontend is built and served through the same nginx
instance.
