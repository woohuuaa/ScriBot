## Design Principles

### UUID Primary Keys

All entities use UUID primary keys for distributed system compatibility and security.

### Soft Deletes

Critical data uses soft deletes via Laravel's `SoftDeletes` trait to preserve historical
information.

### User Tracking

Content modifications track user responsibility through `created_by` and `updated_by` fields.

## Core Entities

### Departments

Organizational departments that users belong to and create topics for.

**Key Fields:**

- `id` (UUID) - Primary key
- `name` (VARCHAR 256) - Department name
- `short_name` (VARCHAR 20) - Abbreviation
- `is_deleted` (BOOLEAN) - Legacy delete flag
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

### Rooms

Physical or virtual rooms where debates take place.

**Key Fields:**

- `id` (UUID) - Primary key
- `name` (VARCHAR 256) - Room name
- `normalized_name` (VARCHAR 256) - Normalized for search
- `is_deleted` (BOOLEAN) - Legacy delete flag
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

### Users

System users with departmental assignments and authentication.

**Key Fields:**

- `id` (UUID) - Primary key
- `name` (VARCHAR 256) - Full name
- `email` (VARCHAR 256) - Unique email address
- `password` (VARCHAR 256) - Hashed password
- `department_id` (UUID) - Foreign key to departments
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

### Roles

System-wide roles for debate participation (e.g., Speaker, Commissioner).

**Key Fields:**

- `id` (UUID) - Primary key
- `name` (VARCHAR 256) - Role name
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

### Department Roles

Department-specific roles with custom naming.

**Key Fields:**

- `id` (UUID) - Primary key
- `name` (VARCHAR 256) - Role name
- `department_id` (UUID) - Foreign key to departments
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

### Debates

Main entity representing parliamentary debate sessions.

**Key Fields:**

- `id` (UUID) - Primary key
- `title` (VARCHAR 256) - Debate title
- `description` (TEXT) - Detailed description
- `date` (DATETIME) - Debate date and time
- `room_id` (UUID) - Foreign key to rooms
- `transcribe_status` (VARCHAR 50) - Transcription state
- `status` (VARCHAR 50) - Overall debate state
- `preparation` (TEXT) - AI-generated preparation
- `preparation_questions` (JSON) - Preparation questions
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

**Constraints:**

- Unique constraint on (date, room_id)
- Foreign key to rooms with CASCADE delete

### Debate User Roles (Pivot)

Links users to debates with specific roles.

**Key Fields:**

- `id` (UUID) - Primary key
- `debate_id` (UUID) - Foreign key to debates
- `user_id` (UUID) - Foreign key to users
- `role` (VARCHAR 256) - Role name (string-based)
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

**Constraints:**

- Foreign keys to debates and users with CASCADE delete

### Debate Departments (Pivot)

Links departments to debates for multi-department participation.

**Key Fields:**

- `debate_id` (UUID) - Foreign key to debates
- `department_id` (UUID) - Foreign key to departments
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

**Constraints:**

- Composite primary key on (debate_id, department_id)
- Foreign keys with CASCADE delete

### Topics

Categorization system for debate content and questions.

**Key Fields:**

- `id` (UUID) - Primary key
- `name` (VARCHAR 256) - Topic name
- `department_id` (UUID) - Foreign key to departments
- `created_by` (UUID) - Foreign key to users
- `is_deleted` (BOOLEAN) - Legacy delete flag
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

**Constraints:**

- Foreign key to departments with CASCADE delete
- Foreign key to users with SET NULL delete
- Index on (department_id, is_deleted)

### Debate Topics (Pivot)

Links topics to debates for categorization.

**Key Fields:**

- `debate_id` (UUID) - Foreign key to debates
- `topic_id` (UUID) - Foreign key to topics
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

**Constraints:**

- Composite primary key on (debate_id, topic_id)
- Foreign keys with CASCADE delete

### Debate Questions

Questions extracted from debate transcripts with metadata.

**Key Fields:**

- `id` (UUID) - Primary key
- `debate_id` (UUID) - Foreign key to debates
- `text` (VARCHAR 2048) - Question text
- `question_number` (unsignedInteger) - Sequential number
- `segment_id` (VARCHAR 256) - Reference to transcript segment
- `topic_id` (UUID) - Foreign key to topics
- `party` (VARCHAR 256) - Political party
- `commissioner` (VARCHAR 256) - Question commissioner
- `assigned_to` (VARCHAR 256) - Assigned person
- `timestamp_ms` (bigInteger) - Timestamp in milliseconds
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

**Constraints:**

- Foreign key to debates with CASCADE delete
- Foreign key to topics with SET NULL delete
- Index on (debate_id, question_number)

### Debate Topics (Pivot)

```sql
CREATE TABLE debate_topics (
    debate_id UUID NOT NULL,
    topic_id UUID NOT NULL,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,

    PRIMARY KEY (debate_id, topic_id),
    FOREIGN KEY (debate_id) REFERENCES debates(id) ON DELETE CASCADE,
    FOREIGN KEY (topic_id) REFERENCES topics(id) ON DELETE CASCADE
);
```

### Debate Questions

```sql
CREATE TABLE debate_questions (
    id UUID PRIMARY KEY,
    debate_id UUID NOT NULL,
    text VARCHAR(2048) NOT NULL,
    question_number unsignedInteger NOT NULL,
    segment_id VARCHAR(256) NULL,
    topic_id UUID NULL,
    party VARCHAR(256) NULL,
    commissioner VARCHAR(256) NULL,
    assigned_to VARCHAR(256) NULL,
    timestamp_ms bigInteger NULL,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    deleted_at TIMESTAMP NULL, -- Soft delete

    FOREIGN KEY (debate_id) REFERENCES debates(id) ON DELETE CASCADE,
    FOREIGN KEY (topic_id) REFERENCES topics(id) ON DELETE SET NULL,
    INDEX(debate_id, question_number)
);
```

### Segments

Individual transcript segments from real-time transcription.

**Key Fields:**

- `id` (UUID) - Primary key
- `debate_id` (UUID) - Foreign key to debates
- `segment_id` (VARCHAR 256) - External segment identifier
- `segment_timestamp` (DATETIME) - Segment timestamp
- `timestamp_ms` (bigInteger) - Millisecond precision
- `original_text` (TEXT) - AI-generated transcript
- `manual_text` (TEXT) - User-corrected text
- `is_checked` (BOOLEAN) - Verification status
- `created_by` (UUID) - Foreign key to users
- `updated_by` (UUID) - Foreign key to users
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

**Constraints:**

- Unique constraint on (debate_id, segment_id)
- Foreign keys with CASCADE/SET NULL delete
- Indexes on segment_timestamp and timestamp_ms

### Transcript Corrections

Global correction rules for transcript processing.

**Key Fields:**

- `id` (BIGINT) - Auto-increment primary key
- `value` (VARCHAR 255) - Text to find (lowercase)
- `replacement` (VARCHAR 255) - Replacement text
- `created_by` (UUID) - Foreign key to users
- `updated_by` (UUID) - Foreign key to users
- `deleted_at` (TIMESTAMP) - Soft delete timestamp

**Constraints:**

- Unique constraint on value
- Foreign keys to users with SET NULL delete

### Debate Files

File attachments for debates (documents, media, etc.).

**Key Fields:**

- `id` (UUID) - Primary key
- `debate_id` (UUID) - Foreign key to debates
- `original_name` (VARCHAR 256) - Original filename
- `file_path` (VARCHAR 512) - Storage path
- `file_size` (BIGINT) - File size in bytes
- `mime_type` (VARCHAR 100) - MIME type
- `uploaded_by` (UUID) - Foreign key to users

**Constraints:**

- Foreign key to debates with CASCADE delete
- Foreign key to users with SET NULL delete

### Transcript Corrections

```sql
CREATE TABLE transcript_corrections (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    value VARCHAR(255) NOT NULL COMMENT 'Original text to find (stored lowercase)',
    replacement VARCHAR(255) NOT NULL COMMENT 'Replacement text (case-sensitive)',
    created_by UUID NULL,
    updated_by UUID NULL,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    deleted_at TIMESTAMP NULL, -- Soft delete

    UNIQUE(value),
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE SET NULL,
    FOREIGN KEY (updated_by) REFERENCES users(id) ON DELETE SET NULL
);
```

### Debate Files

```sql
CREATE TABLE debate_files (
    id UUID PRIMARY KEY,
    debate_id UUID NOT NULL,
    original_name VARCHAR(256) NOT NULL,
    file_path VARCHAR(256) NOT NULL,
    mime_type VARCHAR(256) NOT NULL,
    file_size BIGINT UNSIGNED NOT NULL,
    uploaded_by UUID NOT NULL,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,

    FOREIGN KEY (debate_id) REFERENCES debates(id) ON DELETE CASCADE,
    FOREIGN KEY (uploaded_by) REFERENCES users(id) ON DELETE CASCADE,
    INDEX(debate_id, uploaded_by)
);
```

## Key Relationships Summary

| Entity                     | Related To   | Relationship Type                           | Description |
| -------------------------- | ------------ | ------------------------------------------- | ----------- |
| **Users** → Departments    | Many-to-One  | Users belong to one department              |
| **Debates** → Rooms        | Many-to-One  | Many debates occur in one room              |
| **Debates** ↔ Users       | Many-to-Many | Via `debate_user_roles` with roles          |
| **Debates** ↔ Departments | Many-to-Many | Via `debate_departments`                    |
| **Debates** → Segments     | One-to-Many  | One debate has many transcript segments     |
| **Segments** → Questions   | One-to-Many  | Questions reference specific segments       |
| **Topics** ↔ Debates      | Many-to-Many | Categorization via `debate_topics`          |
| **Users** → Topics         | One-to-Many  | Users create topics within their department |

## Indexing Strategy

- **Composite Indexes**: `(debate_id, segment_id)`, `(debate_id, question_number)`
- **Foreign Key Indexes**: Automatic on all foreign key columns
- **Performance Indexes**: `timestamp_ms` for time-based queries
- **Unique Constraints**: Prevent duplicate data and ensure data integrity

## Data Flow Patterns

1. **Debate Creation**: Department users create debates in specific rooms
2. **Participant Assignment**: Users assigned roles via `debate_user_roles`
3. **Transcription**: Audio segments stored with edit tracking
4. **Question Extraction**: AI questions linked to topics and segments
5. **Corrections**: Global rules improve transcript quality
